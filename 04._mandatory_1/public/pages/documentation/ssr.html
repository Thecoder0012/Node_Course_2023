Server side rendering


When we server side render, we usually use templating engines, and sending variables to the html file.

One can also create their own templating engine.

By reading the html paths and rendering them.
To read them you will need the filesystem import

            
                import fs from "fs";

                function render(page) {
                const navbar = fs
                    .readFileSync("./public/components/navbar/navbar.html")
                    .toString();
                return navbar + page;
                }

In the above code, we see that when a page is rendered, it will have the navbar with it returned.

This is a better practise than creating navbars every time in all the html files.


To read a page, you should use the readFileSync method, and make it into a string.

            
            function read(pagePath) {
                return fs.readFileSync(pagePath).toString();
                }

Example of rendering a specific page:

            
                function renderDocsPage() {
                const path = read("./public/pages/documentation/documentation.html");
                const page = render(path);
                return page;
}
